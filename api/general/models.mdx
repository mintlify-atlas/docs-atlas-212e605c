The `general_superstaq.models` module provides Pydantic data models used for communication with the Superstaq server. These models ensure type safety and validation for API requests and responses.

## Enumerations

### JobType

The different types of jobs that can be submitted through Superstaq.

<ResponseField name="SUBMIT" type="str">
  A job that involves submitting circuits to an external device. Value: `"submit"`
</ResponseField>

<ResponseField name="SIMULATE" type="str">
  A job that requires superstaq to simulate circuits. Value: `"simulate"`
</ResponseField>

<ResponseField name="COMPILE" type="str">
  A job that only involves compiling a circuit for some given device. Value: `"compile"`
</ResponseField>

<ResponseField name="CONVERT" type="str">
  A job that only involves converting circuits between e.g. qiskit and cirq. Value: `"convert"`
</ResponseField>

```python
from general_superstaq.models import JobType

job_type = JobType.SUBMIT
print(job_type.value)  # "submit"
```

### SimMethod

The different simulation methods that are available.

<ResponseField name="NOISE_SIM" type="str">
  The simulation applies noise. If used with device simulation, this applies realistic device noise; otherwise the noise needs to be provided by the user (via the options dict). Value: `"noise-sim"`
</ResponseField>

<ResponseField name="SIM" type="str">
  Samples the circuits without any noise. Value: `"sim"`
</ResponseField>

```python
from general_superstaq.models import SimMethod

sim_method = SimMethod.SIM
```

### CircuitStatus

The possible statuses of a job/circuit.

<ResponseField name="RECEIVED" type="str">
  The job has been received (and accepted) to the server and is awaiting further action.
</ResponseField>

<ResponseField name="AWAITING_COMPILE" type="str">
  The job is waiting for a worker to compile.
</ResponseField>

<ResponseField name="AWAITING_SUBMISSION" type="str">
  The job is waiting for a worker to submit the circuit to an external device.
</ResponseField>

<ResponseField name="AWAITING_SIMULATION" type="str">
  The job is waiting for a worker to simulate.
</ResponseField>

<ResponseField name="COMPILING" type="str">
  The job is being compiled by a worker.
</ResponseField>

<ResponseField name="RUNNING" type="str">
  The job is currently running on a device. SUBMIT jobs only.
</ResponseField>

<ResponseField name="SIMULATING" type="str">
  The job is currently being simulated. SIMULATE jobs only.
</ResponseField>

<ResponseField name="PENDING" type="str">
  When a job has been submitted and is waiting to be run on a QPU. SUBMIT jobs only.
</ResponseField>

<ResponseField name="FAILED" type="str">
  The job failed. A reason should be stored in the job.
</ResponseField>

<ResponseField name="UNRECOGNIZED" type="str">
  Something has gone wrong! (Treated as terminal)
</ResponseField>

<ResponseField name="COMPLETED" type="str">
  The job is completed.
</ResponseField>

<ResponseField name="CANCELLED" type="str">
  The job was cancelled. A reason should be stored in the job.
</ResponseField>

<ResponseField name="DELETED" type="str">
  When a job has been deleted.
</ResponseField>

```python
from general_superstaq.models import CircuitStatus, TERMINAL_CIRCUIT_STATES

if status in TERMINAL_CIRCUIT_STATES:
    print("Job has finished")
```

## Job models

### NewJob

The data model for submitting new jobs.

<ParamField path="job_type" type="JobType" required>
  The job type.
</ParamField>

<ParamField path="target" type="str" required>
  The target device.
</ParamField>

<ParamField path="circuits" type="str" required>

</ParamField>

</ParamField>

<ParamField path="shots" type="int" default="0">

</ParamField>

<ParamField path="dry_run" type="bool" default="False">

</ParamField>

<ParamField path="sim_method" type="SimMethod | None" default="None">

</ParamField>

<ParamField path="priority" type="int" default="0">

</ParamField>

<ParamField path="options_dict" type="dict[str, Any]" default="{}">
  Options dictionary with additional configuration detail.
</ParamField>

<ParamField path="tags" type="list[str]" default="[]">
  Optional tags.
</ParamField>

<ParamField path="metadata" type="dict[str, Any]" default="{}">
  Additional metadata passed by the user.
</ParamField>

```python
from general_superstaq.models import NewJob, JobType, CircuitType

new_job = NewJob(
    job_type=JobType.SUBMIT,
    target="ibmq_qasm_simulator",
    circuits=serialized_circuits,
    circuit_type=CircuitType.CIRQ,
    shots=100,
    tags=["experiment-1"]
)
```

### JobQuery

The query model for retrieving jobs. Using multiple values in a field is interpreted as logical OR while providing values for multiple fields is interpreted as logical AND.

<ParamField path="user_email" type="list[str] | None" default="None">
  List of user emails to include.
</ParamField>

<ParamField path="job_id" type="list[uuid.UUID] | None" default="None">
  List of job IDs to include.
</ParamField>

<ParamField path="target_name" type="list[str] | None" default="None">
  List of targets to include.
</ParamField>

<ParamField path="status" type="list[CircuitStatus] | None" default="None">
  List of statuses to include.
</ParamField>

<ParamField path="tags" type="list[str] | None" default="None">
  Filter for jobs with a given tag.
</ParamField>

<ParamField path="min_priority" type="int | None" default="None">
  Minimum priority to include.
</ParamField>

<ParamField path="max_priority" type="int | None" default="None">
  Maximum priority to include.
</ParamField>

<ParamField path="submitted_before" type="datetime.datetime | None" default="None">
  Filter for jobs submitted before this date.
</ParamField>

<ParamField path="submitted_after" type="datetime.datetime | None" default="None">
  Filter for jobs submitted after this date.
</ParamField>

```python
from general_superstaq.models import JobQuery, CircuitStatus
import datetime

query = JobQuery(
    status=[CircuitStatus.COMPLETED],
    tags=["experiment-1"],
    submitted_after=datetime.datetime(2026, 1, 1)
)
```

## Base model

### DefaultPydanticModel

Default pydantic model used across the superstaq server. All data models inherit from this base class.

**Configuration:**
- `use_enum_values=True`: Enums are serialized to their values
- `extra="ignore"`: Extra fields are ignored during validation
- `validate_assignment=True`: Field assignments are validated
- `validate_default=True`: Default values are validated

```python
from general_superstaq.models import DefaultPydanticModel

class MyModel(DefaultPydanticModel):
    field1: str
    field2: int = 0
```
