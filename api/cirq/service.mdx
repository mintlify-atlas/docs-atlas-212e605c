The `Service` class provides access to Superstaq's API for quantum circuit execution and compilation.

## Service class

A class to access Superstaq's API with support for various quantum hardware targets and simulators.

### Constructor

```python
Service(
    api_key: str | None = None,
    remote_host: str | None = None,
    default_target: str | None = None,
    api_version: str = "v0.3.0",
    max_retry_seconds: int = 3600,
    verbose: bool = False,
    cq_token: str | None = None,
    ibmq_token: str | None = None,
    ibmq_instance: str | None = None,
    ibmq_channel: str | None = None,
    use_stored_ibmq_credentials: bool = False,
    ibmq_name: str | None = None,
    **kwargs
)
```

<ParamField path="api_key" type="str | None">
  API key for accessing Superstaq. If not provided, checks environment variable `SUPERSTAQ_API_KEY` or configuration files.
</ParamField>

<ParamField path="remote_host" type="str | None">
  URL of the API endpoint. Defaults to environment variable `SUPERSTAQ_REMOTE_HOST` if not specified.
</ParamField>

<ParamField path="default_target" type="str | None">
  Default target device for job submissions. If None, target must be specified in each call.
</ParamField>

<ParamField path="api_version" type="str" default="v0.3.0">
  Version of the API to use. Must be either "v0.2.0" or "v0.3.0".
</ParamField>

<ParamField path="max_retry_seconds" type="int" default="3600">
  Number of seconds to retry API calls. Defaults to one hour.
</ParamField>

<ParamField path="verbose" type="bool" default="False">
  Whether to print status messages on retriable errors.
</ParamField>

<ParamField path="cq_token" type="str | None">
  Token from CQ cloud, required for CQ hardware submissions.
</ParamField>

<ParamField path="ibmq_token" type="str | None">
  IBM Quantum or IBM Cloud token for accessing IBM devices.
</ParamField>

<ParamField path="ibmq_instance" type="str | None">
  Instance to use when running IBM jobs.
</ParamField>

<ParamField path="ibmq_channel" type="str | None">
  Type of IBM account. Must be "ibm_quantum_platform" or "ibm_cloud".
</ParamField>

<ParamField path="use_stored_ibmq_credentials" type="bool" default="False">
  Whether to retrieve IBM credentials from locally saved accounts.
</ParamField>

<ParamField path="ibmq_name" type="str | None">
  Name of the IBM account to retrieve. Defaults to "default-ibm-quantum".
</ParamField>

## Methods

### create_job

Creates a new job to run circuit(s) on Superstaq.

```python
create_job(
    circuits: cirq.AbstractCircuit | Sequence[cirq.AbstractCircuit],
    repetitions: int = 1000,
    target: str | None = None,
    method: str | None = None,
    verbatim: bool = False,
    tag: Sequence[str] | str = (),
    metadata: Mapping[str, object] | None = None,
    **kwargs
) -> Job | JobV3
```

<ParamField path="circuits" type="cirq.AbstractCircuit | Sequence[cirq.AbstractCircuit]">
  Circuit or list of circuits to run.
</ParamField>

<ParamField path="repetitions" type="int" default="1000">
  Number of times to repeat the circuit.
</ParamField>

<ParamField path="target" type="str | None">
  Device or simulator to run the job on.
</ParamField>

<ParamField path="method" type="str | None">
  Optional execution method (e.g., "dry-run").
</ParamField>

<ParamField path="verbatim" type="bool" default="False">
  Run circuits without compilation.
</ParamField>

<ParamField path="tag" type="Sequence[str] | str">
  Identifying tag(s) to associate with the job.
</ParamField>

<ParamField path="metadata" type="Mapping[str, object] | None">
  Additional data to store with the job.
</ParamField>

<ResponseField name="return" type="Job | JobV3">
  A Job object that can be queried for status and results.
</ResponseField>

<ParamField path="repetitions" type="int">
  Number of times to run the circuit(s).
</ParamField>

<ParamField path="target" type="str | None">
  Device to run the job on.
</ParamField>

<ParamField path="param_resolver" type="cirq.ParamResolver">
  Parameter resolver for parameterized circuits.
</ParamField>

<ParamField path="method" type="str | None">

### aqt_compile

Compiles circuit(s) for the Advanced Quantum Testbed (AQT).

```python
aqt_compile(
    circuits: cirq.Circuit | Sequence[cirq.Circuit],
    target: str = "aqt_keysight_qpu",
    *,
    num_eca_circuits: int | None = None,
    random_seed: int | None = None,
    atol: float | None = None,
    gate_defs: Mapping | None = None,
    gateset: Mapping[str, Sequence[Sequence[int]]] | None = None,
    pulses: object = None,
    variables: object = None,
    **kwargs
) -> CompilerOutput
```

<ParamField path="circuits" type="cirq.Circuit | Sequence[cirq.Circuit]">
  Circuit(s) to compile.
</ParamField>

<ParamField path="target" type="str" default="aqt_keysight_qpu">
  AQT device target.
</ParamField>

<ParamField path="num_eca_circuits" type="int | None">
  Number of logically equivalent circuits for Equivalent Circuit Averaging (ECA).
</ParamField>

<ParamField path="random_seed" type="int | None">
  Seed for approximate synthesis and ECA.
</ParamField>

<ParamField path="atol" type="float | None">
  Tolerance for approximate gate synthesis.
</ParamField>

<ParamField path="gate_defs" type="Mapping | None">
  Dictionary mapping qtrl config names to operations (unitary matrix, cirq.Gate, cirq.Operation, or None).
</ParamField>

<ParamField path="gateset" type="Mapping[str, Sequence[Sequence[int]]] | None">
  Gates to use for compilation with qubit indices they act upon.
</ParamField>

<ParamField path="pulses" type="object">
  Qtrl PulseManager or file path for pulse configuration.
</ParamField>

<ParamField path="variables" type="object">
  Qtrl VariableManager or file path for variable configuration.
</ParamField>

<ResponseField name="return" type="CompilerOutput">
  Object with optimized circuit(s) and optional qtrl Sequence for pulse sequences.
</ResponseField>

## Example usage

```python
import cirq
import cirq_superstaq as css

# Initialize service
service = css.Service(api_key="YOUR_API_KEY")

# Create a simple circuit
q0, q1 = cirq.LineQubit.range(2)
circuit = cirq.Circuit(
    cirq.H(q0),
    cirq.CNOT(q0, q1),
    cirq.measure(q0, q1, key='result')
)

# Submit job
job = service.create_job(
    circuit,
    repetitions=1000,
    target="ss_unconstrained_simulator"
)

# Get results
counts = job.counts()
print(counts)
```
